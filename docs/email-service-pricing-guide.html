What MileX Needs

MileX er email mainly transactional email: offer letter send, rate forwarded notification, line manager approval/revision notification, and customer activation confirmation.

Important requirements:

  • Customer email recipient will come from KAM Recommendation form field: Email Address.
  • Internal recipients can be fixed env values: SALES_EMAIL, LINE_MANAGER_EMAIL, KAM_EMAIL.
  • Email should be sent from backend API, not directly from frontend.
  • Sender domain should be verified with SPF/DKIM for better inbox delivery.

Purchase / Setup Process - Mailgun

  1. Go to mailgun.com/pricing.
  2. Click Try for free or Get started under Basic.
  3. Create Mailgun account and verify your email.
  4. Add your sending domain. Recommended sender: no-reply@yourdomain.com.
  5. Mailgun will show DNS records for SPF/DKIM/tracking. Add these records in your domain DNS.
  6. Wait until domain status becomes verified.
  7. Create an API key from Mailgun dashboard.
  8. Add environment variables in your app/server:
    • MAILGUN_API_KEY
    • MAILGUN_DOMAIN
    • EMAIL_FROM=no-reply@yourdomain.com
  9. In MileX backend, connect Generate & Email Offer Letter to Mailgun API.
  10. Send first test email to your own Gmail and check Inbox/Spam.

Purchase / Setup Process - Postmark

  1. Go to postmarkapp.com/pricing.
  2. Click Start free trial.
  3. Create account and verify your email.
  4. Create a Server for MileX transactional emails.
  5. Add a Sender Signature or verify a full domain. Domain verification is better.
  6. Add SPF/DKIM DNS records shown by Postmark.
  7. Create or copy Server API Token.
  8. Add environment variables:
    • POSTMARK_SERVER_TOKEN
    • EMAIL_FROM=no-reply@yourdomain.com
  9. Connect MileX backend email route with Postmark API.
  10. Test offer letter email and check delivery logs in Postmark Activity.

Purchase / Setup Process - Resend

  1. Go to resend.com/pricing.
  2. Create account and verify email.
  3. Add your sending domain.
  4. Add DNS records for SPF/DKIM.
  5. Create API key.
  6. Add env variables:
    • RESEND_API_KEY
    • EMAIL_FROM=no-reply@yourdomain.com
  7. Use backend API to send customer offer letter emails.

Purchase / Setup Process - SendGrid

  1. Go to sendgrid.com/en-us/pricing.
  2. Create account and choose Free Trial or Essentials plan.
  3. Complete sender/domain authentication.
  4. Add SPF/DKIM DNS records.
  5. Create API key with Mail Send permission.
  6. Add env variables:
    • SENDGRID_API_KEY
    • EMAIL_FROM=no-reply@yourdomain.com
  7. Connect backend API to SendGrid Mail Send API.

Purchase / Setup Process - Amazon SES

  1. Create AWS account at aws.amazon.com/ses.
  2. Open Amazon SES console and choose a region.
  3. Verify your sending domain.
  4. Add SPF/DKIM DNS records in your domain DNS.
  5. Request production access because new SES accounts usually start in sandbox mode.
  6. Create IAM user or access keys with SES send permission.
  7. Add env variables:
    • AWS_ACCESS_KEY_ID
    • AWS_SECRET_ACCESS_KEY
    • AWS_REGION
    • EMAIL_FROM=no-reply@yourdomain.com
  8. Connect MileX backend API to Amazon SES.

How Email Will Work in MileX

  1. KAM creates recommendation and enters customer Email Address.
  2. Sales prepares rate and Line Manager approves.
  3. Sales clicks Generate & Email Offer Letter.
  4. MileX backend reads customer email from record.recommendation.emailAddress or record.emailAddress.
  5. Backend sends email through Mailgun/Postmark API.
  6. Database stores sent status: sentTo, sentAt, providerMessageId, and status OFFER DELIVERED (PENDING AGREEMENT).
